To perform an exploratory factor analysis, you can use the function fa of the package psych. You will have to specify the data, and the columns that you want to include in the factor analyses. Furthermore, you will have to specify the number of factors that you want to extract, the rotation method and the estimation method.

The possible rotation methods that you can use are:

rotation_methods <- c("none", "varimax", "quartimax", "bentlerT", "equamax", "varimin",
                        "geominT", "bifactor", "promax", "oblimin", "simplimax", "bentlerQ",
                        "geominQ", "biquartimin", "cluster")

While the possible estimation methods are:

estimation_methods <- c("minres", "ols", "wls", "gls", "pa", "ml", "minchi",
                        "minrank", "old.min", "alpha")

To perform an exploratory factor analysis on the trust variables, you can use the following code to get started.

Please make sure to remove the {brackets} if you want to run the analysis. Other symbols should remain in the code.

library(psych)
library(GPArotation)

efa_trust <- fa(df[, {the columns of interest}], nfactors = {the number of factors to extract}, rotate = "{rotation method}", fm = "{estimation method}")

In order to determine the number of factors to extract, you might want to look at the eigenvalues of the factors or the scree plot. These information can be accessed by using the following code:

plot({efa_object_name}$values, type = "b")
{efa_object_name}$values

For the analysis regarding the attitude variables, you could use the following code:

library(psych)
library(GPArotation)

efa_att <- fa(df[, {the columns of interest}], nfactors = {the number of factors to extract}, rotate = "{rotation method}", fm = "{estimation method}")

And for information about the number of factors, you can use the following commands.

{efa_object_name}$values
plot({efa_object_name}$values, type = "b")

7.0.4 Question 4

Apart from the number of factors, you also want to look at the factor loadings. They can be found in the âpattern matrixâ. The higher the factor loadings are, the more indicative an item is for the latent factor. If you find some items to have only very low loadings (indicating that the items do not provide much information about the factor), you may choose not to include them in your analysis. This means you have to rerun the analysis under question 3.

Click for explanation You can find the factor loadings by means of the âprintâ-function used in question 3, or you can search for the variable âloadingsâ, which is inside the results object, to end up with just the information you are searching for.

{efa_object_name}$loadings

And to suppress factor loadings lower than .30, you can use the following code.

print({efa_object_name}$loadings, cut = .30, digits = 2)

Furthermore, if you want to perform a factor analysis without, say, stfedu, while you want all other variables included in your factor analysis, you can simply leave the column number of stfedu, which is 13, unmentioned as follows:

efa_trust_without_stfedu <- fa(df[, c(7:12, 14:19)], nfactors = {the number of factors to extract}, rotate = "{rotation method}", fm = "{estimation method}")
